publishFlow

fun SnsClient.publishFlow(upstream: <Error class: unknown class><PublishMessageRequest>, concurrency: Int = 1, groupStrategy: <Error class: unknown class> = GroupStrategy.TimeWindow(10, 250.milliseconds), topicArn: suspend () -> String): <Error class: unknown class><PublishMessageResponse>

The publishFlow function is used to publish messages concurrently to an Amazon Simple Notification Service (SNS) topic using a SnsClient.

It takes an input Flow of PublishMessageRequest and publishes the messages concurrently, respecting the specified groupStrategy.

The groupStrategy parameter defines how the messages should be grouped. Using a time window, for instance, the messages are going to be grouped either when the maximum number of items is reached, or when the time duration has elapsed, whichever comes first. This can help balance between processing latency and the granularity of data aggregation, leading to more efficient processing by potentially reducing the number of requests to AWS.

Return

A Flow of PublishMessageResponses for the published messages.

Example usage:

val snsClient: SnsClient = ...
val messages: Flow<PublishMessageRequest> = ...

val responses: Flow<PublishMessageResponse> = snsClient.publishFlow(topicArn, messages) {
    "arn:aws:sns:us-east-1:123456789012:MyTopic"
}

responses.collect { response ->
    when (response) {
        is Successful -> println("Message ${response.id} successfully published with messageId: ${response.messageId}")
        is Failure -> println("Message ${response.id} failed to publish with error code: ${response.code}")
    }
}

Parameters

upstream

The input Flow of PublishMessageRequests to be published.

The degree of concurrency to use for publishing messages (default is 1).

The GroupStrategy to use for grouping the messages (default is TimeWindow with 10 items and 250 milliseconds).

A lambda function that returns the Amazon Resource Name (ARN) of the SNS topic to publish the messages to.
